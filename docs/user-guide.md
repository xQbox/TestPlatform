## Описание проекта
Friendly Verter - это наш вызов Angry Verter, разработанный для автоматической проверки студенческих работ в рамках учебного курса школы. Он запускает набор unit-тестов, проверяет код на утечки памяти, стилевые ошибки. База тестов постоянно пополняется, а в ряды борьбы с Angry Verter вступают все новые и новые contibutor'ы.
Наша цель сделать описание ошибок более понятным для всех студентов!
# Важно!

1) Возможна ситуация, что Friendly Verter (наше тестирование) пройдет успешно, а наш Angry Verter(тестирование не пройдет) сломается на одном из кейсов - это нормально!!!
Мы постоянно расширяем базу тестов, готовим их описание и работаем над улучшением! Стараемся быть лучше для Вас!

2) Возможна ситуация, что Angry Verter (тестирование школы) выдаст 100%, а наш Friendly сломается на одном из кейсов - это нормально!!! В наших рядах умнейшие ребята потоков.
Мы постоянно расширяем базу тестов, готовим их описание и работаем над улучшением! Стараемся быть лучше для Вас!


## Установка (пример для дня D07T04)

1) Склонируйте репозиторий
```bash
git clone git@github.com:xQbox/TestPlatform.git 
```

2) Перейдите в папку 
```bash
 cd TestPlatform/
```

3) Скачайте все ветки
```bash
git fetch --all
```

4) Переключитесь на ветку интересующего дня (нейминг веток эквивалентен неймингу веток школы) 
```bash
git checkout D07T04
```

5) Скопируйте файлы .gitignore, run_tests.sh и папку tests/ в корень проекта выбранного дня
```bash
cp  -r {.gitignore,run_tests.sh,tests} ./../D07T04 
```
6) Скопируйте Makefile в папку src проекта выбранного дня
```bash
cp  Makefile ./../D07T04/src/ 
```

## Использование 
### ВАЖНО! после запуска make test (Пункт N2) - к вашим файлам применится .clang-format школы, что позволит отныне и впредь проходить проверки vertera на .clang-format
1)  make start
- Создание/переключение ветки с любой на develop  
- Копирует .clang-format 
- добавляет .gitignore, чтобы при отправке в ветку игнорировалась локальная система тестирования
- 
```bash
make start 
```
2) make test
- запускает тесты 

```bash
make test 
```

## Умеем 
- Проверять ваш код на граничные случаи (Методология классов эквивалентности)
- Проверять ваш код на функции, содержащие больше 32 строк
- Проверять ваш код под valgrind для проверки утечек памяти
- Проверять ваш код с флаго address sanitizer 
- Проверять ваш код с помощью cppcheck

## Не умеем
- ПОКА ЧТО нет текстового описания тест кейсов с объяснением; можно лишь посмотреть какой тест не проходит!

- Проверять ваш код на стилистику (snake_case, camelCase, PascalCase)
- Проверять ваш код на читинг (дата изменения, частота сохранения)
- Проверять на 7 принципов структурного программирования
